## Discussion topics

A pool to draw from — vote/prioritise on arrival.

### Software & algorithms

- **CHIMERYS** ([Nature Methods 2025](https://www.nature.com/articles/s41592-025-02663-w)) — spectrum-centric, DDA/DIA/PRM-agnostic deconvolution; is anyone using it? How does it compare to DIA-NN / FragPipe on our data?
- **DIA-NN 2.0** — new library-free modes, updated FDR model; has everyone migrated? Pain points?
- **De novo sequencing** — deep learning methods (e.g. [DiNovo](https://www.nature.com/articles/s41467-026-70224-6)); practical use cases in a core facility?
- **AlphaFold 3 / structure-informed search** — any routine use for protein inference or PTM localisation?

### Benchmarking & QC

- **[ProteoBench](https://www.biorxiv.org/content/10.64898/2025.12.09.692895v2.full)** — community platform for comparing workflows; are we contributing data? Should we set up a shared C4L submission?
- **QC pipelines** — state of QCloud/QSample, rawDiag, PTXQC; do we have compatible QC metrics across sites?
- **FDR revisited** — ML-based FDR (Percolator, DIA-NN internal) vs classical target–decoy; do we trust them equally?

### Clinical & large-scale proteomics

- **Plasma proteomics at scale** — Astral, Evosep, large cohorts; storage, compute, LIMS coupling
- **Multi-omics integration** — what does the core actually deliver: just the protein table, or linked to RNA/genome?

### Reproducibility & delivery

- **Workflow managers** — Nextflow vs Snakemake vs Galaxy; which direction is each site going?
- **Customer reports** — HTML vs PDF, interactive (iSEE, ExploreDE) vs static; sharing practice across sites
- **SDRF / FAIR metadata** — how much do we enforce locally; barriers to PRIDE submission

### AI tooling

- **LLMs in analysis** — AI-assisted report writing, parameter tuning, literature triage; what are we actually using vs. what is hype?
- **Vibe-coding for omics** ([PMC 2026](https://pmc.ncbi.nlm.nih.gov/articles/PMC12888021/)) — AI-generated analysis apps; risk vs. productivity
